# SignalsArtifact — porting the six-signal pack from RaidUI (v2b)

**What this is:** the C# port of the per-pull diagnostic signal pack the last pre-port UI ran on —
the backbone of RaidUI's DiagStrip and CoverageTimeline. Six per-second timelines + snap markers +
whole-pull aggregates, computed from a `PullReplay` walk. This is the first item from the
2026-06-09 unported-math audit, and the v2b rung of the live-insight ladder: the between-pull card
gains healer-coverage and spacing evidence.

**The operative spec is one JS function:** `buildPullSignals` in
`D:\work\RaidUI\app\server.cjs` (lines ~554-854) — the server side of
`GET /api/pulls/:id/signals`, whose response shape is typed in
`D:\work\RaidUI\NewUI\app\src\api\signals.ts`. The older `src/diagnostics/signals/*.js` files are
the Track-H classify substrate (different consumers, richer coverage model); this port follows the
server function, which is what the shipped UI actually displayed. The classify-grade coverage
model (per-healer centroids, offender attribution — `src/shape/coverage-timeline.js`) is v2c.

## The six signals (1-second buckets over the replay)

All positions are normalized [0,1] × `ArenaYd` for yards. Alive = `EntityHp > 0.001`
(HP defaults to 1 when a frame carries no HP). Players only (`Kind == "Player"`); healers =
`Role` contains "heal" (already populated on `ReplayEntity` by Tempo's replay builder).

| Signal | Per frame | Bucket | Null sentinel |
|---|---|---|---|
| spacing | pairwise avg yards across alive players | mean of valid frames | <2 alive all second → null |
| coverage | fraction of alive players within `healerRangeYd` (30) of any alive healer; a healer is covered by another healer in range, or trivially when it is the only healer alive | mean | — |
| deathsPerSec | — | count of `UnitDied` events by `PullTimeMs` | — |
| followership | magnitude of mean per-player drift vs prior frame, ×50, capped 1 | mean | — |
| entropy | pop-stddev of per-player drift magnitudes, ×50, capped 1 | mean | — |
| hpVariance | pop-stddev of alive players' HP fractions | mean | — |

**Snaps:** coverage drop ≥ 0.10 within a ≤3s lookahead vs the prior second; duplicates within 3s
suppressed. **Aggregates:** coverageAvg, coverageMin, spacingTightest (min of per-second min
pairwise distance where >0), fragileSec (seconds with coverage < 0.6), snapCount, deathsTotal,
durationSec. **Peaks:** coverage and spacing report their MIN (worst moment); the rest MAX.

## C# shape

- `SignalsArtifact.BuildForReplay(PullReplay, healerRangeYd=30) → PullSignalsDto` — the pure port,
  unit-testable on synthetic replays.
- `SignalsArtifact.BuildJson(ParseResult, json)` — per-night artifact (one card per boss, one
  signals block per pull with movement), cached as `.signals.v1.json` beside the other artifacts,
  regenerated by the parse step's existing if-ANY-missing guard.
- `GET /api/signals?name=` — mirrors `/api/trends`.
- **Live evidence:** `TryBuildCoordinationText` gains a `HEALER COVERAGE & SPACING` section — one
  line per pull (coverage avg/min-at, fragile seconds, snap count + biggest, tightest spacing),
  computed from the same between-pull parse. The card starts saying "coverage bottomed at 41%
  right before the second death wave."

## Parity discipline

The JS function has no committed fixtures of its own (it was verified live); parity here =
**invariant tests in xUnit on synthetic replays**, asserting the rules above (solo-healer
coverage, the <2-alive spacing null gate, snap threshold + dedup, stddev math, deaths bucketing),
plus an eyeball pass of real values from the 2026-06-09 M+ log against plausibility (coverage near
1.0 for a coordinated 5-man kill, spacing in single-digit yards on stack phases).
